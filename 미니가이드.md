# 택갈이코드 미니가이드

TUI 기반 CLI AI 채팅 앱 사용법 요약. 처음 쓰는 분은 **설치 → 첫 실행 → 슬래시 커맨드** 순으로 읽으면 5분 안에 다 익힐 수 있습니다.

---

## 1. 설치

플랫폼에 맞는 바이너리를 다운로드해서 실행 권한을 주고 PATH에 넣으면 끝.

### 배포 세트 3종

| 세트 | 연결 대상 | 기본 모델 (Super / Dev) | 바이너리 이름 | 설정 디렉터리 |
|------|-----------|--------------------------|----------------|----------------|
| **기본 (Novita)** | `api.novita.ai/openai` | `openai/gpt-oss-120b` / `qwen/qwen3-coder-30b` | `techai-{platform}` | `~/.tgc` |
| **사내망 (Shinhan)** | `techai-web-prod.shinhan.com/v1` | 동일 (gpt-oss-120b / qwen3-coder-30b) | `techai-onprem-{platform}` | `~/.tgc-onprem` |
| **Gemma 전용 (Novita)** | `api.novita.ai/openai` | `google/gemma-4-31b-it` (Super/Dev 동일) | `techai-gemma-{platform}` | `~/.tgc-gemma` |

설정 디렉터리가 달라서 **세 바이너리가 서로 간섭하지 않고 동시에 설치 가능**합니다. 예를 들어 사내망에서 onprem을 쓰고, 퇴근 후엔 집에서 gemma로 실험할 수 있습니다.

### 플랫폼별 파일명

```
techai-darwin-arm64         (Mac Apple Silicon)
techai-darwin-amd64         (Mac Intel)
techai-windows-amd64.exe    (Windows 64bit)
techai-linux-amd64          (Linux x64)
techai-linux-arm64          (Linux ARM)
```

세트별로 `techai-` 뒤에 `onprem-` 또는 `gemma-` 접두어가 붙을 뿐 구조는 같습니다.

---

## 2. 첫 실행

```bash
chmod +x techai-darwin-arm64
./techai-darwin-arm64
```

첫 실행 시 API 키 입력 프롬프트가 뜹니다. 그 이후는 곧바로 채팅 UI로 진입합니다.

### 기본 조작

| 키 | 동작 |
|---|---|
| `Enter` | 메시지 전송 |
| `Shift+Enter` | 줄바꿈 |
| `Tab` | 모드 전환 (Super ↔ Dev ↔ Plan) |
| `Ctrl+C` / `Esc` | 스트리밍 중지 또는 앱 종료 |
| `Cmd+V` / `Ctrl+V` | 붙여넣기 (10줄 초과 시 `[15줄]` 형태로 요약 표시) |
| 마우스 휠 | 대화 히스토리 스크롤 |

### HUD(상태바) 읽는 법

화면 하단 상태바에는 현재 세션 정보가 한 줄로 요약되어 있습니다.

```
 Super  gpt-oss-120b  ./myproject  ⎇ main*  [DEBUG]  Tool:ON(7)  1247tok  ctx:12%  4.3s
```

| 요소 | 의미 |
|------|------|
| `Super` | 현재 모드 (Super / Dev / Plan). 모드마다 시스템 프롬프트가 다름 |
| `gpt-oss-120b` | 활성 모델 (공급자 접두어 제거 후 표시) |
| `./myproject` | 작업 디렉터리 이름 |
| `⎇ main*` | Git 브랜치 + dirty 표시. `*`가 붙으면 커밋되지 않은 변경이 있음. 초록=clean, 노랑=dirty |
| `[DEBUG]` | 디버그 모드 빌드 표시 (v0.4.0부터 항상 표시됨) |
| `Tool:ON(7)` | 툴 콜 활성화 + 현재 모드에서 사용 가능한 툴 수 |
| `1247tok` | 현재 대화 누적 토큰 |
| `ctx:12%` | 모델 컨텍스트 윈도우 대비 사용률. 70%↑ 노랑, 80%↑ 빨강, **90%↑에서 자동 LLM 요약 트리거** |
| `4.3s` | 마지막 응답 소요 시간 |

## 3. 슬래시 커맨드 전체 목록

메시지 입력창에 `/`로 시작하는 명령을 입력하고 `Enter` 누르면 실행됩니다. 대소문자 구분합니다.

### `/help`
현재 사용 가능한 모든 커맨드와 기본 단축키를 한 줄 요약으로 보여줍니다. 까먹었을 때 가장 먼저 치는 명령.

```
Enter — 전송    Shift+Enter — 줄바꿈    Tab — 모드 전환
/new — 새 세션    /sessions — 목록    /session <id> — 복원
/git — 저장소 상태    /clear — 화면 정리    /setup — API 키    Ctrl+C — 종료
```

---

### `/setup`
API 키를 다시 입력하는 설정 화면으로 진입합니다. 키가 만료됐거나 엔드포인트를 바꾸고 싶을 때 사용. 입력 후 `Enter`로 저장하면 즉시 반영됩니다.

**저장 위치:**
- 기본: `~/.tgc/config.yaml`
- 사내망: `~/.tgc-onprem/config.yaml`
- Gemma: `~/.tgc-gemma/config.yaml`

---

### `/clear`
화면(뷰포트)과 현재 대화 히스토리를 초기화합니다. **시스템 프롬프트는 유지**되므로 모드는 그대로. 페르소나는 남기되 대화만 처음부터 다시 시작하고 싶을 때 사용합니다.

> 주의: 현재 세션의 SQLite 레코드는 지워지지 않습니다. `/sessions`에서 여전히 보입니다. 완전히 새로 시작하려면 `/new`를 쓰세요.

---

### `/new` (v0.6.0 신규)
**새 세션을 생성**합니다. 이전 세션은 DB에 그대로 남고, 화면만 깔끔하게 리셋 + 새 세션 ID 발급. 모드(Super/Dev/Plan)는 유지됩니다.

```
[SESSION] new id=3
```

---

### `/sessions` (v0.6.0 신규)
최근 10개 세션을 최신순으로 나열합니다. 현재 활성 세션에는 `*` 마커가 붙습니다.

```
최근 세션 (/session <id> 로 불러오기)
* #3  untitled                          [openai/gpt-oss-120b]  04-10 14:22
  #2  Phase 1 구현 완료 커밋 메시지 ..  [openai/gpt-oss-120b]  04-10 13:40
  #1  안녕 택갈이 테스트                [openai/gpt-oss-120b]  04-09 22:15
```

**제목은 첫 유저 메시지의 앞 60자**가 자동으로 사용됩니다. 그래서 "안녕 테스트"로 시작하면 제목도 그렇게 됩니다. 의미있게 시작하면 나중에 찾기 편합니다.

---

### `/session <id>` (v0.6.0 신규)
저장된 세션을 **완전히 복원**합니다. 히스토리, 모드, 메시지 스트림까지 그대로 돌아옵니다.

```
[SESSION] #2 'Phase 1 구현 완료 커밋 메시지 작성' 복원됨 (14 메시지)
```

**사용 시나리오:**
- 어제 하던 디버깅을 오늘 이어서 하고 싶음
- 실수로 터미널을 닫아서 대화가 사라짐
- 이전 시도의 컨텍스트를 다시 보고 싶음

**복원 후 동작:**
- 유저/어시스턴트 메시지, tool call 결과까지 모두 재렌더
- 모드는 저장 당시 모드로 복귀
- 토큰 카운터는 리셋됨 (다음 응답 시 provider usage로 다시 채워짐)

---

### `/git` (v0.6.0 신규)
현재 작업 디렉터리의 git 상태를 요약해서 보여줍니다.

**저장소인 경우:**
```
git: main
staged:    2
unstaged:  5
untracked: 1
```

**클린한 경우:**
```
git: main
clean working tree
```

**저장소가 아닌 경우:**
```
git: 현재 디렉토리는 git 저장소가 아닙니다.
```

> HUD 상단의 `⎇ main*` 라벨은 앱이 캐시된 스냅샷을 보여주지만, `/git`은 **매번 새로 `git status`를 실행**합니다. 최신 상태를 확인하고 싶을 때 사용하세요. (500ms 타임아웃, 비-git 디렉터리에서도 블로킹 없음)

## 4. 모드 전환 (Tab 키)

`Tab`으로 세 가지 모드를 순환합니다. 모드마다 시스템 프롬프트와 사용 가능한 툴이 다릅니다.

| 모드 | 색상 | 용도 | 주요 툴 |
|------|------|------|---------|
| **Super** | 주황 | 범용 대화, 설명, 리서치 | 기본 툴 (shell_exec, knowledge 등) |
| **Dev** | 파랑 | 실제 코드 작성/편집 | 파일 편집 툴 포함 |
| **Plan** | 초록 | 분석, 설계, 계획 수립 | read-only 툴 위주 |

모드 전환 시 화면 상단에 현재 모드 정보 박스가 다시 출력됩니다.

## 5. 자동 기능 (조작할 필요 없음)

이런 것들은 자동으로 동작하므로 그냥 알고만 있으면 됩니다.

### 컨텍스트 자동 관리
- 매 턴 시작 전에 오래된 툴 결과를 **Snip/Micro** 단계로 눌러서 토큰 절약
- `ctx:XX%`가 **90%를 넘으면 자동으로 LLM 요약**을 호출해 windows 절반 크기로 압축
- 모델 윈도우는 자동 감지 (gpt-oss-120b=128K, qwen3-coder-30b=256K, gemma-4-31b=128K)

### 세션 자동 저장
- 앱 시작 시 `~/.tgc/sessions.db` (또는 onprem/gemma 변형)에 자동 저장소 오픈
- 모든 메시지(유저, 어시스턴트, tool call/result)가 실시간으로 DB에 append됨
- 첫 유저 메시지가 세션 제목으로 자동 사용
- 실패해도 silent fallback (메모리 모드로 계속 작동)

### Git 상태 자동 갱신
- 부팅 시 1회
- 매 스트림 시작 직전
- 툴 결과 수신 직후 (편집 반영)

### 스트리밍 안정성
- 100ms 틱으로 스피너/타이머 실시간 갱신
- 5초 지연 시 `⚠ 응답지연`, 15초 무응답 시 `⚠ 응답없음`
- 60초 무응답 시 최대 3회 자동 재시도 (이전 응답 보존)
- 디버그 모드에서 엔드포인트 tool calling 지원 자동 점검

## 6. 환경 변수 (선택)

config 파일 대신 환경 변수로 일회성 오버라이드도 가능합니다.

```bash
export TGC_API_KEY=sk-...            # API 키 (/setup 대신)
export TGC_BASE_URL=https://...      # 엔드포인트 변경
export TGC_MODEL_SUPER=...           # Super 모드 모델
export TGC_MODEL_DEV=...             # Dev 모드 모델
```

주로 CI나 원격 실험 환경에서 유용합니다.

## 7. 트러블슈팅 자주 있는 질문

| 증상 | 해결 |
|------|------|
| 첫 실행 시 API 키 입력이 안 뜸 | `rm -rf ~/.tgc*` 후 재실행 |
| `/sessions`가 "저장소가 열려 있지 않습니다"라고만 함 | `~/.tgc*/sessions.db` 권한 확인. 홈 디렉터리가 쓰기 가능한지 체크 |
| HUD에 `⎇ main`이 안 나옴 | 현재 디렉터리가 git 저장소가 아님. 정상 동작 |
| `Tool:OFF`로만 보임 | 엔드포인트가 tool calling을 지원하지 않음. `/setup`에서 다른 엔드포인트 사용 |
| `ctx:<1%`로만 표시됨 | 정상. 토큰이 매우 적을 때 정수 나눗셈 결과가 0이라서 이렇게 표시됨 |
| 사내망에서 연결 실패 | 사내망 빌드는 `techai-onprem-*`을 사용해야 함. 일반 빌드는 Novita로 나감 |

## 8. 커맨드 치트시트 (한 화면 요약)

```
# 기본 조작
Enter          메시지 전송
Shift+Enter    줄바꿈
Tab            모드 전환 (Super → Dev → Plan)
Ctrl+C / Esc   스트리밍 중지 / 앱 종료

# 슬래시 커맨드
/help                 도움말
/setup                API 키 재설정
/clear                화면 + 히스토리 초기화 (세션 유지)
/new                  새 세션 생성
/sessions             최근 10개 세션 목록
/session <id>         특정 세션 복원
/git                  저장소 상태 요약
```
