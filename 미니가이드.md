# 택갈이코드 미니가이드

TUI 기반 CLI AI 채팅 앱 사용법 요약. 처음 쓰는 분은 **설치 → 첫 실행 → 슬래시 커맨드** 순으로 읽으면 5분 안에 다 익힐 수 있습니다.

---

## 1. 설치

플랫폼에 맞는 바이너리를 다운로드해서 실행 권한을 주고 PATH에 넣으면 끝.

### 배포 세트 3종

| 세트 | 연결 대상 | 기본 모델 (Super / Dev) | 바이너리 이름 | 설정 디렉터리 |
|------|-----------|--------------------------|----------------|----------------|
| **기본 (Novita)** | `api.novita.ai/openai` | `openai/gpt-oss-120b` / `qwen/qwen3-coder-30b` | `techai-{platform}` | `~/.tgc` |
| **사내망 (Shinhan)** | `techai-web-prod.shinhan.com/v1` | 동일 (gpt-oss-120b / qwen3-coder-30b) | `techai-onprem-{platform}` | `~/.tgc-onprem` |
| **Gemma 전용 (Novita)** | `api.novita.ai/openai` | `google/gemma-4-31b-it` (Super/Dev 동일) | `techai-gemma-{platform}` | `~/.tgc-gemma` |

설정 디렉터리가 달라서 **세 바이너리가 서로 간섭하지 않고 동시에 설치 가능**합니다. 예를 들어 사내망에서 onprem을 쓰고, 퇴근 후엔 집에서 gemma로 실험할 수 있습니다.

### 플랫폼별 파일명

```
techai-darwin-arm64         (Mac Apple Silicon)
techai-darwin-amd64         (Mac Intel)
techai-windows-amd64.exe    (Windows 64bit)
techai-linux-amd64          (Linux x64)
techai-linux-arm64          (Linux ARM)
```

세트별로 `techai-` 뒤에 `onprem-` 또는 `gemma-` 접두어가 붙을 뿐 구조는 같습니다.

---

## 2. 첫 실행

```bash
chmod +x techai-darwin-arm64
./techai-darwin-arm64
```

첫 실행 시 API 키 입력 프롬프트가 뜹니다. 그 이후는 곧바로 채팅 UI로 진입합니다.

### 기본 조작

| 키 | 동작 |
|---|---|
| `Enter` | 메시지 전송 |
| `Shift+Enter` | 줄바꿈 |
| `Tab` | 모드 전환 (Super ↔ Dev ↔ Plan) |
| `Ctrl+C` / `Esc` | 스트리밍 중지 또는 앱 종료 |
| `Cmd+V` / `Ctrl+V` | 붙여넣기 (10줄 초과 시 `[15줄]` 형태로 요약 표시) |
| 마우스 휠 | 대화 히스토리 스크롤 |

### HUD(상태바) 읽는 법

화면 하단 상태바에는 현재 세션 정보가 한 줄로 요약되어 있습니다.

```
 Super  gpt-oss-120b  ./myproject  ⎇ main*  [DEBUG]  Tool:ON(7)  1247tok  ctx:12%  4.3s
```

| 요소 | 의미 |
|------|------|
| `Super` | 현재 모드 (Super / Dev / Plan). 모드마다 시스템 프롬프트가 다름 |
| `gpt-oss-120b` | 활성 모델 (공급자 접두어 제거 후 표시) |
| `./myproject` | 작업 디렉터리 이름 |
| `⎇ main*` | Git 브랜치 + dirty 표시. `*`가 붙으면 커밋되지 않은 변경이 있음. 초록=clean, 노랑=dirty |
| `[DEBUG]` | 디버그 모드 빌드 표시 (v0.4.0부터 항상 표시됨) |
| `Tool:ON(7)` | 툴 콜 활성화 + 현재 모드에서 사용 가능한 툴 수 |
| `1247tok` | 현재 대화 누적 토큰 |
| `ctx:12%` | 모델 컨텍스트 윈도우 대비 사용률. 70%↑ 노랑, 80%↑ 빨강, **90%↑에서 자동 LLM 요약 트리거** |
| `4.3s` | 마지막 응답 소요 시간 |

## 3. 슬래시 커맨드 전체 목록

메시지 입력창에 `/`로 시작하는 명령을 입력하고 `Enter` 누르면 실행됩니다. 대소문자 구분합니다.

### `/help`
현재 사용 가능한 모든 커맨드와 기본 단축키를 한 줄 요약으로 보여줍니다. 까먹었을 때 가장 먼저 치는 명령.

```
Enter — 전송    Shift+Enter — 줄바꿈    Tab — 모드 전환
/new — 새 세션    /sessions — 목록    /session <id> — 복원
/git — 저장소 상태    /clear — 화면 정리    /setup — API 키    Ctrl+C — 종료
```

---

### `/setup`
API 키를 다시 입력하는 설정 화면으로 진입합니다. 키가 만료됐거나 엔드포인트를 바꾸고 싶을 때 사용. 입력 후 `Enter`로 저장하면 즉시 반영됩니다.

**저장 위치:**
- 기본: `~/.tgc/config.yaml`
- 사내망: `~/.tgc-onprem/config.yaml`
- Gemma: `~/.tgc-gemma/config.yaml`

---

### `/clear`
화면(뷰포트)과 현재 대화 히스토리를 초기화합니다. **시스템 프롬프트는 유지**되므로 모드는 그대로. 페르소나는 남기되 대화만 처음부터 다시 시작하고 싶을 때 사용합니다.

> 주의: 현재 세션의 SQLite 레코드는 지워지지 않습니다. `/sessions`에서 여전히 보입니다. 완전히 새로 시작하려면 `/new`를 쓰세요.

---

### `/new` (v0.6.0 신규)
**새 세션을 생성**합니다. 이전 세션은 DB에 그대로 남고, 화면만 깔끔하게 리셋 + 새 세션 ID 발급. 모드(Super/Dev/Plan)는 유지됩니다.

```
[SESSION] new id=3
```

---

### `/sessions` (v0.6.0 신규)
최근 10개 세션을 최신순으로 나열합니다. 현재 활성 세션에는 `*` 마커가 붙습니다.

```
최근 세션 (/session <id> 로 불러오기)
* #3  untitled                          [openai/gpt-oss-120b]  04-10 14:22
  #2  Phase 1 구현 완료 커밋 메시지 ..  [openai/gpt-oss-120b]  04-10 13:40
  #1  안녕 택갈이 테스트                [openai/gpt-oss-120b]  04-09 22:15
```

**제목은 첫 유저 메시지의 앞 60자**가 자동으로 사용됩니다. 그래서 "안녕 테스트"로 시작하면 제목도 그렇게 됩니다. 의미있게 시작하면 나중에 찾기 편합니다.

---

### `/session <id>` (v0.6.0 신규)
저장된 세션을 **완전히 복원**합니다. 히스토리, 모드, 메시지 스트림까지 그대로 돌아옵니다.

```
[SESSION] #2 'Phase 1 구현 완료 커밋 메시지 작성' 복원됨 (14 메시지)
```

**사용 시나리오:**
- 어제 하던 디버깅을 오늘 이어서 하고 싶음
- 실수로 터미널을 닫아서 대화가 사라짐
- 이전 시도의 컨텍스트를 다시 보고 싶음

**복원 후 동작:**
- 유저/어시스턴트 메시지, tool call 결과까지 모두 재렌더
- 모드는 저장 당시 모드로 복귀
- 토큰 카운터는 리셋됨 (다음 응답 시 provider usage로 다시 채워짐)

---

### `/git` (v0.6.0 신규)
현재 작업 디렉터리의 git 상태를 요약해서 보여줍니다.

**저장소인 경우:**
```
git: main
staged:    2
unstaged:  5
untracked: 1
```

**클린한 경우:**
```
git: main
clean working tree
```

**저장소가 아닌 경우:**
```
git: 현재 디렉토리는 git 저장소가 아닙니다.
```

> HUD 상단의 `⎇ main*` 라벨은 앱이 캐시된 스냅샷을 보여주지만, `/git`은 **매번 새로 `git status`를 실행**합니다. 최신 상태를 확인하고 싶을 때 사용하세요. (500ms 타임아웃, 비-git 디렉터리에서도 블로킹 없음)

## 4. 모드 전환 (Tab 키)

`Tab`으로 세 가지 모드를 순환합니다. 모드마다 시스템 프롬프트와 사용 가능한 툴이 다릅니다.

| 모드 | 색상 | 용도 | 주요 툴 |
|------|------|------|---------|
| **Super** | 주황 | 범용 대화, 설명, 리서치 | 기본 툴 (shell_exec, knowledge 등) |
| **Dev** | 파랑 | 실제 코드 작성/편집 | 파일 편집 툴 포함 |
| **Plan** | 초록 | 분석, 설계, 계획 수립 | read-only 툴 위주 |

모드 전환 시 화면 상단에 현재 모드 정보 박스가 다시 출력됩니다.

## 5. 자동 기능 (조작할 필요 없음)

이런 것들은 자동으로 동작하므로 그냥 알고만 있으면 됩니다.

### 컨텍스트 자동 관리
- 매 턴 시작 전에 오래된 툴 결과를 **Snip/Micro** 단계로 눌러서 토큰 절약
- `ctx:XX%`가 **90%를 넘으면 자동으로 LLM 요약**을 호출해 windows 절반 크기로 압축
- 모델 윈도우는 자동 감지 (gpt-oss-120b=128K, qwen3-coder-30b=256K, gemma-4-31b=128K)

### 세션 자동 저장
- 앱 시작 시 `~/.tgc/sessions.db` (또는 onprem/gemma 변형)에 자동 저장소 오픈
- 모든 메시지(유저, 어시스턴트, tool call/result)가 실시간으로 DB에 append됨
- 첫 유저 메시지가 세션 제목으로 자동 사용
- 실패해도 silent fallback (메모리 모드로 계속 작동)

### Git 상태 자동 갱신
- 부팅 시 1회
- 매 스트림 시작 직전
- 툴 결과 수신 직후 (편집 반영)

### 스트리밍 안정성
- 100ms 틱으로 스피너/타이머 실시간 갱신
- 5초 지연 시 `⚠ 응답지연`, 15초 무응답 시 `⚠ 응답없음`
- 60초 무응답 시 최대 3회 자동 재시도 (이전 응답 보존)
- 디버그 모드에서 엔드포인트 tool calling 지원 자동 점검

## 6. 환경 변수 (선택)

config 파일 대신 환경 변수로 일회성 오버라이드도 가능합니다.

```bash
export TGC_API_KEY=sk-...            # API 키 (/setup 대신)

export TGC_MODEL_SUPER=...           # Super 모드 모델

```

주로 CI나 원격 실험 환경에서 유용합니다.

## 7. 트러블슈팅 자주 있는 질문

| 증상 | 해결 |
|------|------|
| 첫 실행 시 API 키 입력이 안 뜸 | `rm -rf ~/.tgc*` 후 재실행 |
| `/sessions`가 "저장소가 열려 있지 않습니다"라고만 함 | `~/.tgc*/sessions.db` 권한 확인. 홈 디렉터리가 쓰기 가능한지 체크 |
| HUD에 `⎇ main`이 안 나옴 | 현재 디렉터리가 git 저장소가 아님. 정상 동작 |
| `Tool:OFF`로만 보임 | 엔드포인트가 tool calling을 지원하지 않음. `/setup`에서 다른 엔드포인트 사용 |
| `ctx:<1%`로만 표시됨 | 정상. 토큰이 매우 적을 때 정수 나눗셈 결과가 0이라서 이렇게 표시됨 |
| 사내망에서 연결 실패 | 사내망 빌드는 `techai-onprem-*`을 사용해야 함. 일반 빌드는 Novita로 나감 |

## 8. 플랫폼별 동작 (Mac / Linux / Windows)

같은 바이너리라도 OS마다 저장 위치나 의존성이 조금씩 다릅니다. 처음 설치할 때 한 번 훑어보면 나중에 이상할 때 원인을 빠르게 짚을 수 있습니다.

### 8.1 설정 & 세션 DB 위치

모두 Go의 `os.UserHomeDir()`을 기준으로 홈 디렉터리를 찾고, 그 아래 `ConfigDirName`(`.tgc` / `.tgc-onprem` / `.tgc-gemma`) 폴더를 만듭니다.

| OS | 홈 디렉터리 | 기본 세트 경로 | 전체 파일 구조 |
|----|--------------|-----------------|-----------------|
| **macOS** | `/Users/<username>` | `/Users/<username>/.tgc/` | `config.yaml` + `sessions.db` (+ WAL/shm) |
| **Linux** | `/home/<username>` | `/home/<username>/.tgc/` | 동일 |
| **Windows** | `C:\Users\<username>` | `C:\Users\<username>\.tgc\` | 동일 (파일 구조는 같음) |

세트별로 폴더만 다르지 구조는 완전히 같습니다.

```
~/.tgc/
├── config.yaml        # API 키, 엔드포인트, 모델 설정
├── sessions.db        # SQLite 본체 (모든 대화 저장)
├── sessions.db-wal    # WAL 저널 (스트리밍 중 임시)
└── sessions.db-shm    # 공유 메모리 인덱스
```

> Windows에서 홈 디렉터리가 OneDrive로 리다이렉트된 경우 `C:\Users\<u>\OneDrive\...` 밑에 만들어질 수 있습니다. OneDrive 동기화 중 락 충돌이 날 수 있으니, 가능하면 `OneDrive` 밖으로 옮기거나 `TGC_CONFIG_DIR`(환경 변수)로 오버라이드하세요.

### 8.2 SQLite 세션 영속성

`modernc.org/sqlite`(순수 Go 포팅)를 사용하므로 **CGO / 별도 SQLite 설치 불필요**. OS 별 차이점만 정리:

| 항목 | macOS | Linux | Windows |
|------|-------|-------|---------|
| 저널 모드 | WAL | WAL | WAL |
| 파일 락 | `flock` (정상) | `flock` (정상) | `LockFileEx` (정상) |
| 동시 실행 2개 | 가능 (읽기 공유, 쓰기 직렬화) | 가능 | 가능 |
| 권한 이슈 | 거의 없음 | NFS 홈에서 느릴 수 있음 | 백신이 WAL 잠금 가능 |

**공통 동작:**
- 앱 시작 시 파일이 없으면 자동 생성
- 메시지마다 즉시 `append` → 강제 종료해도 마지막 메시지까지 보존
- 열기 실패 시 silent fallback → 메모리 모드로 계속 작동 (`/sessions`만 비어 보임)

### 8.3 Git 연동 (HUD `⎇` + `/git`)

내부적으로 `git status --porcelain=v1 --branch`와 `git rev-parse --abbrev-ref HEAD`를 호출합니다. 즉 **Git 바이너리가 PATH에 있어야 동작**합니다.

| OS | Git 설치 | HUD 라벨 | `/git` |
|----|---------|-----------|--------|
| **macOS** | Xcode CLT 또는 `brew install git` | 자동 | 자동 |
| **Linux** | `apt/yum/pacman install git` | 자동 | 자동 |
| **Windows** | [Git for Windows](https://git-scm.com/download/win) 필수 | Git for Windows 설치 시만 | 동일 |

- Git이 없으면 HUD의 `⎇` 라벨은 그냥 숨겨지고 `/git`은 "git 저장소가 아닙니다"로 표시됩니다. **오류 메시지는 뜨지 않습니다** (500ms 타임아웃 + silent fallback).
- Windows의 경우 Git for Windows가 설치한 `git.exe`의 경로가 PATH에 들어있는지 `where git`으로 확인하세요.
- 경로 구분자는 Go의 `filepath`가 알아서 처리하므로 `/`든 `\`든 상관없습니다.

### 8.4 터미널 UI (Bubble Tea v2)

| OS | 권장 터미널 | Alt Screen | 유니코드 (`⎇ Tool:ON`) | 마우스 휠 |
|----|-----------|-------------|-------------------------|-----------|
| **macOS** | Terminal.app / iTerm2 / Ghostty | ✅ | ✅ (SF Mono 권장) | ✅ |
| **Linux** | GNOME Terminal / Alacritty / Kitty | ✅ | ✅ | ✅ |
| **Windows** | **Windows Terminal 권장** | ✅ WT / ❌ 구 `cmd.exe` | WT만 정상 | WT만 정상 |

**Windows 주의:**
- **구 `cmd.exe`나 PowerShell ISE에서는 컬러/유니코드가 깨질 수 있습니다.** 반드시 [Windows Terminal](https://aka.ms/terminal)을 사용하세요.
- 폰트는 Cascadia Code나 D2Coding처럼 글꼴 링크가 한글+유니코드 글리프를 모두 커버하는 것이 좋습니다.
- WSL2에서도 잘 돌아갑니다 (그 경우 Linux 바이너리 사용).

### 8.5 컨텍스트 자동 압축

OS 무관하게 완전히 동일하게 동작합니다. 모델 윈도우 감지 → `ctx:XX%` 계산 → 90%↑ LLM 요약 호출은 순수 Go 로직입니다.

### 8.6 API 키 저장

`config.yaml`에 **평문 YAML**로 저장됩니다. OS 내장 키체인은 사용하지 않으므로 본인이 권한을 잘 조여두는 것이 좋습니다.

| OS | 권장 권한 | 명령 |
|----|-----------|------|
| **macOS** | `600` | `chmod 600 ~/.tgc/config.yaml` |
| **Linux** | `600` | `chmod 600 ~/.tgc/config.yaml` |
| **Windows** | NTFS ACL (본인만 읽기/쓰기) | `icacls "%USERPROFILE%\.tgc\config.yaml" /inheritance:r /grant:r "%USERNAME%:F"` |

> 팀 공유 PC나 공용 Mac에서는 `TGC_API_KEY` 환경 변수로만 넣고 config에는 저장하지 않는 방법도 있습니다.

### 8.7 스트리밍 (SSE + 자동 재시도)

OS 무관 동일. 다만 TLS 인증서 저장소는 OS 기본을 따릅니다:

- **macOS**: Keychain
- **Linux**: `/etc/ssl/certs/ca-certificates.crt`
- **Windows**: Windows Certificate Store

사내망 프록시/SSL 인터셉션 환경에서 연결 실패 시 해당 OS의 루트 CA에 사내 인증서를 추가해야 합니다.

### 8.8 슬래시 커맨드 동작 차이

**완전히 OS 무관하게 동일**합니다. `/help`, `/setup`, `/clear`, `/new`, `/sessions`, `/session <id>`, `/git` 모두 같은 코드 경로를 탑니다. 유일한 차이는 `/git`이 외부 `git` 바이너리에 의존한다는 점뿐입니다.

### 8.9 플랫폼별 한줄 주의사항

- 🍎 **macOS**: 그냥 동작합니다. 최초 실행 시 Gatekeeper 경고가 뜨면 `xattr -d com.apple.quarantine ./techai-darwin-arm64`로 제거.
- 🐧 **Linux**: NFS 홈 디렉터리는 SQLite WAL이 느려질 수 있습니다. `TGC_CONFIG_DIR=/var/tmp/tgc` 같은 로컬 디스크로 옮기면 빠릅니다.
- 🪟 **Windows**: ① Windows Terminal ② Git for Windows ③ OneDrive 밖 경로 — 이 세 가지만 지키면 Mac/Linux와 동일한 경험을 얻습니다.

### 8.10 설치 확인 치트시트

```bash
# macOS / Linux
ls -la ~/.tgc/
file ./techai-darwin-arm64      # Mac 아키텍처 확인
./techai-darwin-arm64 --version

# Windows (PowerShell)
dir $HOME\.tgc
Get-Item .\techai-windows-amd64.exe | Select VersionInfo
.\techai-windows-amd64.exe --version
```

모든 OS에서 첫 실행 후 `ls`/`dir`로 `sessions.db`가 생기면 설치 성공입니다.
