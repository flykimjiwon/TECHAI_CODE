# SuperSOL 택가이코드 시연 가이드

> 신한은행 사내 시연용. GPT-OSS-120B 모델 기준 최적화.
> UI 시나리오 3개 + 쿼리 시나리오 1개 (미정).

---

## 사전 준비

### 1. 터미널 2개 열기

```bash
# 터미널 1: 웹 서버
cd ~/Desktop/demo-supersol
npm run dev
# → http://localhost:3000 에서 SuperSOL 화면 확인
```

```bash
# 터미널 2: 택가이코드
cd ~/Desktop/demo-supersol
techai
```

### 2. 화면 배치 (녹화용)

```
┌───────────────────────┬───────────────────────┐
│                       │                       │
│     택가이코드         │      브라우저          │
│     (터미널)           │   localhost:3000      │
│                       │                       │
│   프롬프트 입력 →      │   ← 실시간 UI 변화    │
│                       │                       │
└───────────────────────┴───────────────────────┘
```

- 택가이코드가 코드 수정 → Next.js HMR 자동 감지 → 브라우저 즉시 반영
- 별도 새로고침 불필요

### 3. 시나리오 전환

택가이코드에서 `/clear` 입력 → 채팅 정리 후 다음 시나리오 진행.

## 시나리오 1: 홈 — 계좌 카드에 최근 거래 미리보기 추가

### 프롬프트

```
src/views/HomePage.tsx의 은행 계좌 카드 아래에 최근 거래내역 3건 미리보기를 추가해줘. 거래 데이터는 src/data/mock.ts의 transactions 배열을 사용하고, 거래 항목은 src/components/TransactionItem.tsx 컴포넌트를 사용해
```

### 현재 상태

- `HomePage.tsx`에서 `transactions`를 **import하지 않고 있음**
- `mock.ts`에 거래내역 15건이 존재함
- AI가 두 파일을 읽고 → import 추가 → JSX 삽입

### AI 예상 행동

| 단계 | 도구 | 행동 |
|------|------|------|
| 1 | `file_read` | `src/views/HomePage.tsx` 읽기 |
| 2 | `file_read` | `src/data/mock.ts` → transactions 데이터 확인 |
| 3 | `file_edit` | import에 `transactions` 추가 |
| 4 | `file_edit` | 은행 카드 아래에 최근 3건 렌더링 JSX 삽입 |

### 브라우저 변화

## 시나리오 3: 혜택 — 적립목표 포인트 프로그레스바 추가

### 프롬프트

```
src/views/BenefitsPage.tsx의 포인트 카드에 이번 달 적립 목표까지 남은 포인트를 프로그레스바로 보여줘. src/data/mock.ts에 POINT_GOAL 데이터가 있고, 프로그레스바는 src/components/ProgressBar.tsx 컴포넌트를 사용해. 현재 김지원 고객은 2,751P 적립했고 목표는 5,000P야
```

### 현재 상태

- `BenefitsPage.tsx`에서 `CUSTOMER`만 import (points: 2751, membership: PREMIER)
- `mock.ts`에 `MEMBERSHIP_TIERS` 데이터가 존재함 (VIP = 5,000P)
- AI가 멤버십 데이터를 읽고 → import 추가 → 프로그레스바 UI 추가

### AI 예상 행동

| 단계 | 도구 | 행동 |
|------|------|------|
| 1 | `file_read` | `src/views/BenefitsPage.tsx` 읽기 |
| 2 | `file_read` | `src/data/mock.ts` → MEMBERSHIP_TIERS 확인 |
| 3 | `file_edit` | import에 `MEMBERSHIP_TIERS` 추가 |
| 4 | `file_edit` | 포인트 카드 안에 프로그레스바 JSX 삽입 |

### 브라우저 변화 (혜택 탭)

**변경 전:**
```
마이신한포인트
2,751P                [사용하기]
──────────────────────────────
포인트 모으는 중
[미션 2개]  [이벤트 1개]
```

**변경 후:**
```
마이신한포인트
2,751P                [사용하기]
──────────────────────────────
다음 등급: VIP (5,000P)
████████████████░░░░░░ 55%
2,249P 더 모으면 VIP!
──────────────────────────────
포인트 모으는 중
[미션 2개]  [이벤트 1개]
```

### 120B 성공률: 85%+
계산: (2751 - 2000) / (5000 - 2000) = 25% ... 아닌데, 2751/5000 = 55%.
AI가 계산 방식을 약간 다르게 할 수 있지만 프로그레스바 자체는 잘 그릴 것.

## 비상 대응

| 상황 | 대처 |
|------|------|
| AI가 파일을 못 찾음 | 프롬프트에 이미 경로가 있으므로 거의 발생 안 함 |
| 파일 탐색 루프 | Ctrl+C로 중단 후 "src/data/mock.ts 읽어봐" 직접 지시 |
| 수정 위치가 틀림 | "은행 카드 div 닫는 태그 바로 위에 넣어줘" 재요청 |
| HMR 미작동 | 브라우저 Cmd+R 새로고침 |
| 빈 응답 반복 | 자동 재시도 3회 → 실패 시 안내 메시지 표시됨. 프롬프트를 더 짧게 재입력 |
| 빌드 에러 | "에러 확인하고 고쳐줘" 입력 |
