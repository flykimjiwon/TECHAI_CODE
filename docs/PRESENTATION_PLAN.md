# 택가이코드 발표 플랜

> 청중: 사내 부장/임원/본부장/그룹장

## 슬라이드 구성 (9장)

### 1. Title — 택가이코드
- ASCII 로고 재현 (TECHAI CODE)
- 부제: "Go 단일 바이너리 AI 코딩 어시스턴트"
- 발표자명 / 날짜
- **시간: 15초**

### 2. Content — 왜 만들었는가?
- 기존 도구의 한계:
  - Node.js / Python 런타임 필요
  - 특정 모델(Claude, GPT-4 등)에 종속
  - 사내망(개발망) 사용 불가
- 핵심 메시지: "이 3가지를 한번에 해결하고 싶었습니다"
- **시간: 30초**

### 3. Card Grid — 핵심 기능 4가지
- ① **3모드**: Super(만능) / Deep Agent(자율코딩 100회) / Plan(계획) — Tab 키 전환
- ② **멀티에이전트**: GPT-OSS-120B + Qwen3-Coder-30B 병렬 (Review/Consensus/Scan)
- ③ **81개 내장 지식 RAG**: 벡터DB 없이 ~1ms, BXM·Spring·React·DB·Shell 등
- ④ **파일 스냅샷 + /undo**: 수정 전 자동 백업, 즉시 복구
- **시간: 40초**

### 4. Comparison — 기존 오픈소스 vs 택가이코드
| 항목 | 기존 도구 (Aider, Continue 등) | 택가이코드 |
|------|-------------------------------|-----------|
| 배포 | Node/Python 필요 | 단일 바이너리 21MB |
| 모델 | 특정 API 종속 | OpenAI-compatible면 뭐든 OK |
| 멀티에이전트 | 단일 모델 | 2모델 병렬 (3전략) |
| 파일 편집 | 정확 매칭만 | 4단계 Fuzzy 매칭 |
| 사내망 | 별도 설정 필요 | 온프레미스 빌드 즉시 반입 |
| 지식 | 없거나 벡터DB | 81개 내장 + 사용자 문서 |
- **시간: 40초**

### 5. Flow/Process — 모델 갈아끼우기 구조
- 흐름도: `OpenAI-compatible API` → 모델 교체만으로 업그레이드
  - Novita: GPT-OSS-120B / Qwen3-Coder-30B
  - 사내망: techai-web-prod.shinhan.com
  - Gemma: google/gemma-4-31b-it
- 핵심 메시지: "모델이 좋아지면 좋아질수록, 갈아끼우기만 하면 됩니다"
- 3종 빌드가 서로 간섭 없이 동시 설치 가능
- **시간: 30초**

### 6. Content — 강점
- ① **직접 무한 개선 가능**: Go 37,000줄, 71커밋, 소스 완전 소유
- ② **모델 진화 = 자동 업그레이드**: Gemma4 같은 고성능 소형 모델 나올수록 강해짐
- ③ **외부 의존성 제로**: ~21MB, Node/Python/Docker 불필요
- ④ **사내망 즉시 반입 완료**: 이미 사용 중
- **시간: 30초**

### 7. Content — 로드맵
- **택가이 데스크톱**: GUI 앱 (Electron/Tauri 등)
- 향후 확장 방향 예고
- **시간: 20초**

### 8. Section Divider — LIVE DEMO
- "직접 보여드리겠습니다"
- 반전 예고 분위기
- **시간: 10초**

### 9. Closing — 감사합니다

## 데모 시나리오

### Part 1 — 녹화 영상 (인터넷망)

사전에 녹화할 장면들:

1. **터미널 실행**: `techai` → ASCII 로고 + HUD 상태바 등장
2. **모드 전환**: Tab 키로 Super → Deep Agent → Plan 순환
3. **프로젝트 분석**: "이 프로젝트 구조 분석해줘" → list_files + file_read 도구 자동 호출 → 마크다운 결과
4. **프로젝트 초기화**: `/init` → `.techai.md` 자동 생성
5. **멀티에이전트**: `/multi review` → 두 모델이 병렬로 동작하는 장면

### Part 2 — 실시간 라이브 (개발망, 반전!)

**연출**: "사실 이미 사내망에 반입되어 있습니다"
→ 개발망 컴퓨터 접속 → `techai` 실행

**실시간 질문 리스트** (짧고 임팩트 있는 것):

| # | 질문 | 왜 이걸 보여주나 |
|---|------|-----------------|
| 1 | "현재 컴퓨터 IP 주소 뭐야" | shell_exec 도구 호출 → 즉시 답변 (도구 사용 시연) |
| 2 | "현재 디렉토리에 파일 몇 개야?" | list_files → 빠른 응답 |
| 3 | "Spring Security에서 JWT 인증 어떻게 해?" | knowledge_search → 내장 지식 81개 중 자동 매칭 (**핵심 차별점**) |
| 4 | `/git` | 사내 레포 git 상태 즉시 표시 |
| 5 | "이 프로젝트 README.md 요약해줘" | file_read + 요약 (실무 활용 시연) |

> **질문 3이 하이라이트** — "벡터DB 없이 ~1ms로 전문 지식을 찾아서 답변합니다"
