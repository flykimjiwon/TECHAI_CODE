## DeepSeek V4 Flash 모델 스펙

| 항목 | 스펙 |
|------|------|
| 전체 파라미터 | 284B |
| **활성 파라미터** | **13B** (추론 시 MoE) |
| 모델 크기 | **158GB** (FP4+FP8 혼합 정밀도) |
| 컨텍스트 윈도우 | **1M tokens** |
| 라이선스 | **MIT** (상업용 자유) |
| SWE-bench Verified | 79.0% |
| LiveCodeBench | 91.6% |
| 다운로드 | HuggingFace, Ollama, ModelScope |
| 양자화 버전 | 21종 (GGUF Q4/Q5/Q6/Q8 등) |

### 왜 이 모델이 특별한가

- **284B 파라미터인데 13B만 활성** → 대형 모델 성능 + 소형 모델 속도
- **FP4+FP8 혼합** → 158GB로 압축 (일반 284B 모델은 500GB+)
- **MIT 라이선스** → 사내 배포, 수정, 상업 사용 완전 자유
- **1M 컨텍스트** → 전체 코드베이스를 한번에 분석 가능

## H200 5대 서빙 용량 분석

### 기본 계산

| 항목 | 값 |
|------|-----|
| H200 5대 총 VRAM | 141GB × 5 = **705GB** |
| DeepSeek V4 Flash 모델 | **158GB** |
| 모델 로딩 후 여유 VRAM | **547GB** |
| KV 캐시 (1M ctx, 1명) | ~20-30GB |
| KV 캐시 (128K ctx, 1명) | ~3-4GB |

### 동시 사용자 추정

| 컨텍스트 설정 | 동시 사용자 | 적합 용도 |
|:---:|:---:|------|
| **1M** (최대) | **15~20명** | 전체 코드베이스 분석 |
| **256K** | **40~50명** | 대규모 파일 수정 |
| **128K** | **50~80명** | 일반 코딩 작업 |
| **32K** (짧은 대화) | **100명+** | 간단한 질의응답 |

### 실사용 시나리오

실제로는 모든 사용자가 동시에 요청하지 않음.
**동시 접속률 ~30%** 가정 시:

| 전체 사용자 | 동시 접속 (30%) | 필요 컨텍스트 | 가능? |
|:---:|:---:|:---:|:---:|
| 50명 | 15명 | 128K | **충분** |
| 100명 | 30명 | 128K | **가능** |
| 200명 | 60명 | 32K | **가능** |
| 200명 | 60명 | 128K | **부족** (GPU 추가 필요) |

### 서빙 소프트웨어 옵션

| 소프트웨어 | 특징 | 추천도 |
|-----------|------|:---:|
| **vLLM** | PagedAttention, 최고 throughput | ★★★★★ |
| **TGI** (HuggingFace) | 안정적, 쉬운 설정 | ★★★★☆ |
| **SGLang** | 최신, 빠른 배치 처리 | ★★★★☆ |
| **Ollama** | 간단, 소규모 적합 | ★★★☆☆ |

### OpenAI-compatible API

vLLM/TGI 모두 OpenAI 호환 API 제공 → TECHAI CODE의 `base_url`만 변경하면 즉시 연동.

```yaml
# config.yaml — 온프렘 H200 서버
api:
    base_url: https://내부서버:8000/v1
    api_key: (내부 인증키)
models:
    super: deepseek-v4-flash
    dev: deepseek-v4-flash
```

## 양자화 옵션 (GPU 절약)

모델을 양자화하면 VRAM 사용량을 줄일 수 있음:

| 양자화 | 모델 크기 | 성능 손실 | H200 필요 수 |
|--------|----------|----------|:---:|
| FP4+FP8 (원본) | 158GB | 없음 | 2대 |
| Q4_K_M (GGUF) | ~80GB | ~5% | **1대** |
| Q6_K (GGUF) | ~110GB | ~2% | 1대 |
| Q8_0 (GGUF) | ~150GB | ~1% | 2대 |

Q4 양자화 시 **H200 1대로도 모델 로딩 가능** → 나머지 4대는 KV 캐시용 → 동시 사용자 대폭 증가.
